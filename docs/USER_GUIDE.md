## 🧠 ログ学習システムの使用方法

### 基本ワークフロー

#### ステップ1: ログ学習タブの選択
1. **「🧠 ログ学習」**タブをクリック
2. 3つのサブタブから選択：
   - **📁 ファイルアップロード**: 個別ログファイル学習
   - **🗂️ バッチ学習**: 複数ファイル一括学習
   - **📊 学習結果**: 累積統計・概念分析

#### ステップ2: ログファイルの準備
**対応形式:**
- **ChatGPT形式**: `あなた:` / `ChatGPT:` パターン
- **Claude形式**: `Human:` / `Assistant:` パターン  
- **Gemini形式**: `ユーザー:` / `Gemini:` パターン
- **汎用形式**: 段落ベース解析・自動構造化

**推奨ログ特徴:**
- 構造的対話（技術質問・学習ログ・問題解決）
- 専門用語・概念を含む内容
- 質問と回答のペアが明確
- 1,000文字以上（概念抽出精度向上）

#### ステップ3: 学習実行

**📁 ファイルアップロード学習:**
1. 「ファイルを選択」ボタンでログファイル選択
2. 「学習開始」ボタンクリック
3. リアルタイム進捗表示・結果可視化

**🗂️ バッチ学習:**
1. ログファイルを適切なディレクトリに配置
2. 「バッチ学習開始」ボタンクリック
3. 複数ファイル自動スキャン・一括処理

#### ステップ4: 結果確認

**📊 学習統計:**
- 総概念数: 現在の概念DB規模
- 新規概念: 今回学習で追加された概念
- 重複概念: 重複検出・統合された概念
- カテゴリ分布: 7カテゴリ（AI・プログラミング・データサイエンス等）

**🧠 概念詳細:**
- 抽出概念一覧・品質スコア・技術性評価
- カテゴリ分類・関連性分析・推奨事項

## 💡 効果的な使用のコツ

### 1. 高品質ログの準備

**良い例（技術対話）:**
```
Human: TypeScriptでReactコンポーネントの型安全性を高めるための
プロパティ設計について教えてください。ジェネリクスや条件付き型の
活用方法も含めて詳しく知りたいです。

Assistant: TypeScriptでReactコンポーネントの型安全性を高める
プロパティ設計について説明します。ジェネリクスと条件付き型を
活用した実践的なアプローチを示します...
```

**避けるべき例（雑談）:**
```
ユーザー: こんにちは、今日は暑いですね。
ChatGPT: こんにちは！確かに今日は暑いですね。
```

### 2. 学習効果を高める使い方

**推奨頻度:**
- 週1-2回の定期的なログ学習
- 重要な技術対話は即座に学習
- 月1回の品質改善実行

**学習データの蓄積戦略:**
- 専門分野別にログファイルを整理
- 学習成果の高いログパターンを特定
- 継続的な概念DB拡張・品質向上

### 3. トラブルシューティング

**概念抽出結果が少ない場合:**
- ログに専門用語・技術概念が含まれているか確認
- 一般的な雑談は概念抽出対象外の設計
- 質問と回答のペアが明確な構造的対話を使用

**品質改善効果が見えない場合:**
- 重複統合は概念数減少・品質向上が目的
- カテゴリ分類・関係性整理による検索性向上
- 複数回実行で段階的品質向上を期待

**ファイルアップロードエラー:**
- ファイルサイズ制限（通常10MB以下）を確認
- 対応フォーマット（.txt, .md, .log）を使用
- 特殊文字・エンコーディングの問題をチェック
